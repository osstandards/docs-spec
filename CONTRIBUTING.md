# Contributing

Thank you for contributing to `docs-spec`.

## Ways to Contribute

- Improve specification language.
- Add examples.
- Propose new document types.
- Improve templates.
- Report confusing or incomplete sections.

## Contribution Process

1. Open an issue describing the change.
2. Discuss whether the change belongs in the core spec, an example, or an RFC.
3. Submit a pull request.
4. Maintainers review for clarity, compatibility, and practical usefulness.

## Specification Changes

Substantive changes SHOULD use the RFC process in `/rfcs`.
