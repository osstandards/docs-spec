# docs-spec Validation Checklist

Use this checklist to evaluate whether a project follows `docs-spec`.

## Repository Root

- [ ] `README.md` exists.
- [ ] `LICENSE` exists.
- [ ] `CONTRIBUTING.md` exists or contribution instructions are linked.
- [ ] `SECURITY.md` exists or security reporting is documented.
- [ ] Root README links to documentation index.

## Documentation Root

- [ ] `/docs` exists.
- [ ] `docs/README.md` exists.
- [ ] `docs/getting-started.md` exists.
- [ ] Documentation is separated by type.
- [ ] Important docs are not orphaned.

## Quality

- [ ] Internal links work.
- [ ] Setup instructions have been tested.
- [ ] Code examples are current.
- [ ] Changelog or release notes are linked.
- [ ] Documentation ownership is clear.

## Machine Readability

- [ ] `docs-manifest.json` exists.
- [ ] Manifest references valid paths.
- [ ] Manifest uses recognized document types.
